# ta_bench_direct — Direct-Call Benchmark

## Problem
ta_bench measures indicator performance through JSON-RPC servers. The IPC overhead (JSON serialization, pipe reads/writes, dispatch) adds noise. We want a zero-overhead comparison that also naturally tests the real compilation model: reference via libta-lib.a (separate TU, extern globals) vs codegen via single-TU #include (static globals).

## Architecture

Two binaries:

### ta_bench_direct (orchestrator)
- Built by cmake, links libta-lib.a
- Generates deterministic price data (seed=42, same as ta_bench)
- Uses TA_ForEachFunc + TA_CallFunc for direct reference calls
- Times each function: N iterations, 3 passes, keep minimum
- Spawns `ta_bench_cg` with same CLI args, reads its output
- Correlates results, prints comparison table
- CLI: `--function=`, `--iters=`, `--points=` (same as ta_bench)

### ta_bench_cg (codegen direct binary)
- Generated by `ta_codegen generate-bench --backend=c`
- Built by `ta_codegen build`
- #includes all indicator .c files (single TU, static globals)
- Dispatch table: function name string -> direct function call
- Generates same price data (seed=42)
- Runs matching functions, 3 passes minimum, prints `FUNCNAME timing_ns` lines
- No JSON, no IPC on hot path

### regtest.py integration
- Runs ta_bench_direct after existing tests
- `--direct-bench-only` skips regtest + server bench
- Combos with `--no-regtest`

## Output protocol (ta_bench_cg -> ta_bench_direct)
```
FUNCNAME timing_ns
```
One line per function, space-separated. ta_bench_direct reads all lines after spawning, correlates by function name.

## Files to create/modify
- `src/tools/ta_bench/ta_bench_direct.c` — orchestrator
- `tools/ta_codegen/src/bench_gen.rs` — generates ta_bench_cg.c
- `tools/ta_codegen/src/main.rs` — `generate-bench` command + build step
- `CMakeLists.txt` — build ta_bench_direct
- `scripts/regtest.py` — `--direct-bench-only` flag
