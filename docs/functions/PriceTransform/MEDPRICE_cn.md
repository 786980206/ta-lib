# MEDPRICE - 中位价格

## 描述

中位价格是一个简单的价格指标，计算给定周期内最高价和最低价之间的中点。它表示价格范围的中心，对于平滑价格行为并作为其他技术指标的输入很有用。

## 分类
价格变换 / 数学运算符

## 计算方法

中位价格是最高价和最低价的平均值：

### 公式
```
中位价格 = (最高价 + 最低价) / 2
```

## 参数

无

## 输入
- **最高价**：`double[]`
- **最低价**：`double[]`

## 输出
- 中位价格值：`double[]`

## 解释

### 目的
- **范围中点**：高低范围的中心
- **简化价格**：柱范围的单一值
- **支撑/阻力**：可以作为枢轴点
- **平滑**：减少极端高低价的噪音

### 用途
中位价格通常用于：
1. **作为指标输入**：代替收盘价
2. **枢轴计算**：柱的参考点
3. **支撑/阻力**：中点通常很重要
4. **平滑**：中位价格的移动平均线

## 使用示例

```c
// C/C++ 示例
double high[100], low[100];
double medprice[100];
int outBegIdx, outNBElement;

// 计算中位价格
TA_RetCode retCode = TA_MEDPRICE(
    0,                    // 开始索引
    99,                   // 结束索引
    high,                 // 最高价
    low,                  // 最低价
    &outBegIdx,           // 输出：开始索引
    &outNBElement,        // 输出：元素数量
    medprice              // 输出：中位价格值
);
```

## 与其他价格变换的比较

| 价格类型 | 公式 | 特征 |
|----------|------|------|
| 收盘价 | 收盘价 | 最常见，最终价格 |
| 中位价 | (H+L)/2 | 范围中点，忽略收盘价 |
| 典型价 | (H+L+C)/3 | 包括收盘价，平衡 |
| 加权收盘价 | (H+L+C+C)/4 | 强调收盘价 |

## 优势

1. **简单**：非常容易计算
2. **范围焦点**：强调交易范围
3. **平滑**：减少收盘异常的影响
4. **中性**：不受收盘方向偏向

## 限制

1. **忽略收盘价**：不考虑价格结算位置
2. **忽略成交量**：无成交量考虑
3. **简单**：不如典型价或加权价全面

## 相关函数

- **TYPPRICE**：典型价格 (H+L+C)/3
- **WCLPRICE**：加权收盘价 (H+L+C+C)/4
- **AVGPRICE**：平均价格 (O+H+L+C)/4
- **MIDPOINT**：周期中点 - 不同概念
- **MIDPRICE**：周期中点价格 - 使用周期参数

## 附加说明

中位价格在区间市场中特别有用，其中每个柱范围的中点比单独的收盘价提供更清晰的价格平衡图。它经常用于专注于价格范围和波动性而不是方向性收盘的指标中。
