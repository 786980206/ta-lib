# CMO - Chande动量振荡器

## 描述

Chande动量振荡器（CMO）是由Tushar Chande开发的动量指标，用于测量价格变化的动量。与使用平滑收益和损失的RSI不同，CMO使用周期内收益和损失的总和，使其对最近的价格变化更敏感。它在+100和-100之间振荡。

## 分类
动量指标

## 作者
Tushar Chande

## 计算方法

CMO使用周期内收益和损失的总和计算：

### 公式
```
CMO = ((收益总和 - 损失总和) / (收益总和 + 损失总和)) × 100
```

其中：
- 收益总和 = 周期内所有正价格变化的总和
- 损失总和 = 周期内所有负价格变化的总和

### 详细步骤
```
步骤1：计算价格变化（收盘价 - 前收盘价）
步骤2：分离收益（正变化）和损失（负变化）
步骤3：对周期内所有收益求和
步骤4：对周期内所有损失求和
步骤5：应用CMO公式
```

### 示例
对于5周期CMO，变化为：[+2, -1, +3, -2, +1]
- 收益总和 = 2 + 3 + 1 = 6
- 损失总和 = 1 + 2 = 3
- CMO = ((6 - 3) / (6 + 3)) × 100 = (3/9) × 100 = 33.33

## 参数

- **optInTimePeriod**（默认：14）：计算周期
  - 有效范围：2 到 100000
  - 常用值：9、14、20

## 输入
- 价格数据：`double[]`（通常是收盘价）

## 输出
- CMO 数值：`double[]`（范围：-100到+100）

## 解释

### CMO 数值
- **+100**：所有价格变化都是收益（最强上升趋势）
- **-100**：所有价格变化都是损失（最强下降趋势）
- **0**：相等的收益和损失（无动量）
- **+50到+100**：强向上动量
- **-50到-100**：强向下动量
- **-50到+50**：弱动量或横盘

### 交易信号

1. **零线交叉**：
   - **买入**：CMO向上穿越0
   - **卖出**：CMO向下穿越0
   - **最适合**：动量变化检测

2. **超买/超卖**：
   - **超买**：CMO > +50（强向上动量）
   - **超卖**：CMO < -50（强向下动量）
   - **极端**：CMO > +80或< -80

3. **背离**：
   - **看涨**：价格更低低点，CMO更高低点
   - **看跌**：价格更高高点，CMO更低高点
   - **最适合**：趋势衰竭点

4. **趋势强度**：
   - **强上升趋势**：CMO持续为正
   - **强下降趋势**：CMO持续为负
   - **弱趋势**：CMO在零附近振荡

## 使用示例

```c
// C/C++ 示例

int outBegIdx, outNBElement;

```

## 实现细节

TA-Lib CMO 实现：

1. **价格变化**：计算连续收盘价之间的差值
2. **收益/损失分离**：分离正变化和负变化
3. **求和**：对周期内收益和损失求和
4. **CMO公式**：应用动量公式
5. **回望**：需要n个周期才能产生第一个输出

## 交易策略

### 1. 零线策略
- **买入**：CMO向上穿越0
- **卖出**：CMO向下穿越0
- **过滤器**：只有当|CMO| > 25时才交易
- **最适合**：动量变化检测

### 2. 超买/超卖策略
- **买入**：CMO < -50，然后向上穿越-50
- **卖出**：CMO > +50，然后向下穿越+50
- **确认**：等待价格确认
- **最适合**：均值回归市场

### 3. 背离策略
- **设置**：识别价格和CMO之间的背离
- **确认**：等待价格模式
- **入场**：确认时
- **最适合**：趋势衰竭点

### 4. 趋势跟踪策略
- **上升趋势**：CMO > 0，在回调时买入
- **下降趋势**：CMO < 0，在反弹时卖出
- **出场**：CMO穿越零
- **最适合**：趋势市场

## CMO vs. RSI

| 方面 | CMO | RSI |
|------|-----|-----|
| 计算 | 收益/损失总和 | 平滑收益/损失 |
| 响应性 | 更敏感 | 不太敏感 |
| 范围 | -100到+100 | 0到100 |
| 零线 | 是（0） | 否（50） |
| 平滑 | 无 | Wilder平滑 |

## 优势

1. **敏感**：比RSI更敏感
2. **零线**：在0处有清晰的中性点
3. **有界**：总是在-100和+100之间
4. **动量焦点**：直接测量动量
5. **简单**：易于理解和使用

## 限制

1. **噪音**：比RSI更波动
2. **震荡**：可能产生假信号
3. **周期敏感性**：结果随周期选择变化
4. **无平滑**：无平滑的原始动量
5. **震荡市场**：在横盘条件下困难

## 周期选择

### 短期（5-9）
- **特征**：非常敏感，噪音大
- **用途**：日内交易，短期动量
- **权衡**：更多信号，更多噪音
- **最适合**：主动交易

### 标准期（14-20）
- **特征**：平衡响应性
- **用途**：摆动交易，一般分析
- **权衡**：良好平衡
- **最适合**：大多数交易风格

### 长期（25-50）
- **特征**：更平滑，不太敏感
- **用途**：头寸交易，长期趋势
- **权衡**：更少信号，更可靠
- **最适合**：长期分析

## 相关函数

- **RSI**：相对强弱指数 - 类似概念带平滑
- **MOM**：动量 - 绝对变化版本
- **ROC**：变化率 - 百分比变化版本
- **STOCH**：随机振荡器 - 基于区间的动量

## 附加说明

Tushar Chande开发了CMO作为对RSI的改进，通过使用原始总和而不是平滑平均值使其对最近的价格变化更敏感。

### 关键洞察

1. **动量测量**：
   - 价格动量的直接测量
   - 不像RSI那样平滑
   - 对最近变化更敏感
   - 更适合短期分析

2. **零线重要性**：
   - 零 = 相等的收益和损失
   - 穿越零 = 动量变化
   - 距离零的远近 = 动量强度
   - 信号的最重要水平

3. **响应性与平滑性**：
   - CMO：更敏感，更多噪音
   - RSI：不太敏感，更平滑
   - 根据交易风格选择
   - CMO更适合短期交易

4. **最佳应用**：
   - 短期动量分析
   - 趋势变化检测
   - 趋势中的超买/超卖
   - 背离分析

5. **组合策略**：
   - 使用CMO进行动量方向
   - 结合趋势指标
   - 使用成交量确认
   - 多时间框架背景

### 实用技巧

**动量交易**：
- 使用零线交叉进行信号
- 等待价格行为确认
- 使用成交量验证
- 在最近极值之外设置止损

**趋势跟踪**：
- CMO > 0表示上升趋势
- CMO < 0表示下降趋势
- 在趋势确认时在回调时入场
- 当CMO穿越零时出场

**超买/超卖**：
- 使用±50作为阈值
- 等待反转信号
- 用价格模式确认
- 避免在强趋势中使用

**避免震荡**：
- 在趋势市场中使用
- 避免在横盘条件下使用
- 结合趋势过滤器
- 需要其他指标确认

CMO对于需要比RSI更敏感的动量指标的交易者特别有价值。它非常适合短期动量分析，并为动量变化提供清晰的信号。
